# 后端分类规则

## A类 - 纯表面变更（不影响运行逻辑）

- 注释增删
- 代码格式化（缩进、空行、换行风格）
- import 重排序
- 变量/函数重命名（纯重命名，逻辑不变）
- 日志级别调整（log.info → log.debug）
- 配置文件中纯展示性字段修改（如注释、描述字段）
- Swagger/OpenAPI 注释修改（不影响接口行为）
- SQL 中的注释修改
- 类型声明/接口定义增删（仅类型层面，不含运行时逻辑，如 Go interface、Python type hints、Java interface 中无默认实现的方法签名）
- 文档修改（README、CHANGELOG 等）

## B类 - 功能变更（影响运行行为）

- API 接口路径/方法/参数变化
- 业务逻辑修改（if/else 分支、循环条件、算法等）
- 数据库查询/写入变化（SQL 修改、ORM 查询变化）
- 鉴权/授权逻辑变化（中间件、guard、权限检查）
- 数据校验规则变化（参数校验、schema 变化）
- 第三方服务调用变化（支付、推送、邮件、短信等）
- 定时任务/Cron 逻辑变化
- 缓存策略变化（Redis、Memcached 等）
- 事务处理变化
- 错误处理/异常捕获变化
- 环境配置变化（连接字符串、密钥引用、超时设置等）
- 消息队列生产/消费逻辑变化
- import/export 增删（涉及运行时依赖，如新增了第三方包引用）
- DTO/VO/Model 字段增删（影响序列化/反序列化）

## C类 - 混合变更

同一文件同时存在 A类 和 B类

## 后端特有判定注意

- 修改了 DTO/VO 的字段但未检查所有消费方 = B类
- 修改了配置默认值 = B类
- 修改了 SQL 的 WHERE/JOIN 但声称只是优化 = B类（逻辑可能变化）
- 删除了一个 catch 块但保留了 try = B类（异常处理链断裂）
- import 增删：如果只是重命名或移动包路径 = A类；如果引入了新的依赖 = B类
- 宁可误判为 B，不可漏判
