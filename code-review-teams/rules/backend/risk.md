# 后端风险检测规则

## R1 - 逻辑断裂
- 删除了方法/函数/代码块但无替代实现
- 条件分支被删除导致场景未覆盖
- async/await/Promise/ goroutine 修改但错误处理缺失
- try/catch/recover 被删除或 catch 块变空
- return 语句变化导致函数提前退出或不再返回值
- 错误码/错误类型变化但调用方未适配

## R2 - 数据流风险
- DTO/VO/Model 字段变化但调用方未同步
- 接口返回结构变化但前端未适配
- 消息格式/Event Schema 变化但消费方未更新
- 枚举值增删但数据库已有数据未处理
- 序列化/反序列化规则变化（JSON tag、Protobuf 字段号等）

## R3 - 副作用与资源管理遗漏
- 数据库连接/事务未关闭
- 文件句柄/流未关闭
- 锁获取后未释放（分布式锁、互斥锁等）
- 缓存未失效/过期策略变化
- 定时任务/Cron 表达式变化
- 后台任务/Worker 异常退出未处理
- 连接池配置变化

## R5 - API 与鉴权风险
- 接口权限变更（公开→需鉴权，或鉴权→公开）
- Token/Session 验证逻辑变化
- CORS 配置变化
- 速率限制/限流逻辑变化
- API 参数校验规则放宽
- 中间件执行顺序变化
- OAuth/SSO 流程变化

## R6 - 数据层风险
- SQL 查询条件变化（WHERE 子句增删、JOIN 变化）
- 数据库事务边界变化（隔离级别、传播行为）
- 数据迁移脚本中的业务逻辑
- ORM 查询从同步改为异步或反之
- 索引变化影响查询性能
- 批量操作缺少分页/限流
- 软删除/硬删除逻辑变化
- 数据库连接池大小变化

## R7 - 并发与一致性风险
- 共享状态修改但未加锁/未用原子操作
- 缓存与数据库的读写顺序变化
- 分布式场景下的幂等性变化
- 消息队列的消费顺序/重试策略变化
- 并发请求下的竞态条件
- 分布式锁超时时间变化

## R8 - 遗漏联动修改
- 修改了子模块的接口但调用方未同步
- 修改了公共工具函数但所有使用方未排查
- 修改了配置常量但未检查所有引用处
- 修改了接口契约但未通知上下游
- 修改了环境变量/配置但未更新部署配置
- 修改了消息队列的 Topic/Queue 但消费方未同步

## 后端特有检测重点
- **Go**: goroutine 泄漏、channel 未关闭、context 取消逻辑变化、defer 顺序变化
- **Java**: Spring Bean 作用域变化、事务注解变化、线程池配置变化、序列化 ID 变化
- **Python**: 装饰器顺序变化、WSGI/ASGI 中间件变化、Celery 任务签名变化
- **Node.js**: 事件循环阻塞风险、Promise 链断裂、中间件顺序变化、进程管理变化

## 严重程度判定

- **HIGH**: 数据丢失、资金错误、安全漏洞、线上故障级别（R1 逻辑断裂 + R5 鉴权 + R6 数据损坏 + R7 一致性破坏）
- **MEDIUM**: 部分功能异常、体验降级、边缘场景未覆盖（R2 数据流 + R3 资源泄漏 + R6 查询异常）
- **LOW**: 性能轻微下降、代码可维护性风险、非核心路径的边界情况（R8 联动遗漏 + R7 并发边界）
